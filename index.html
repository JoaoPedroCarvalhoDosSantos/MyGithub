Conceitos de GIT e GitHub

O Git possibilita a integração de fluxos de trabalho realizados por vários contribuidores com o passar do tempo, para um determinado repositório. O GitHub é um serviço de hospedagem na Web para repositórios do Git.

Fluxo do Git / GitHub

O fluxo Git/GitHub envolve usar o Git localmente para controlar as versões dos arquivos e, em seguida, usar o GitHub como um repositório remoto para compartilhar e colaborar no código.

Como resolver conflitos

Conflitos acontecem quando duas ou mais pessoas alteram a mesma linha de um arquivo ou fazem modificações incompatíveis no mesmo trecho de código em branches diferentes. O Git não sabe qual alteração manter e solicita que você resolva manualmente. Passos para resolver conflitos: Identificar os arquivos em conflito Após tentar um merge ou pull, o Git exibirá mensagens indicando quais arquivos estão com conflitos. Siga esses passos:

  1. Atualize seu branch: Antes de começar, puxe as últimas alterações do repositório remoto para ter certeza de que seu código está atualizado. Use "git pull"
  2. Inicie a fusão (merge): Ao tentar mesclar um branch que tem conflitos, o Git vai parar o processo e informar quais arquivos estão em conflito.
  3. Identifique e edite os arquivos: Abra os arquivos listados com conflitos. O Git insere marcadores especiais para indicar as diferenças:
      <<<<<<< HEADSeu código (o que você alterou)=======O código que veio do outro branch>>>>>>> nome-do-outro-branch
  4. Resolva o conflito: Remova os marcadores (<<<<<<<, =======, >>>>>>>) e edite o arquivo para que ele fique da forma que você deseja. Você pode escolher a sua versão, a do outro branch, ou combinar as duas.
  5. Adicione e confirme as alterações: Depois de resolver todos os conflitos, adicione os arquivos resolvidos ao "stage" com "git add nome-do-arquivo" e em seguida, faça um commit para finalizar a fusão com "git commit -m "Mensagem do commit""
  6. Envie as alterações (push): Envie o resultado final para o GitHub com "git push"
